# agy — Google Antigravity для Claude Code

Використовуйте **agy CLI** (Google Antigravity) як другу модель усередині Claude Code — це
`agy`-аналог плагіна `codex`. Ставте запитання, отримуйте альтернативні думки, перевіряйте
свій diff або делегуйте завдання зі здатністю записувати — і все це не виходячи з Claude Code.

Що робить його цікавим: **agy може запускати моделі Gemini, Claude *і* GPT-OSS** за єдиним
CLI та обліковим записом. Цей плагін це розкриває — обирайте будь-яку з них для кожного виклику
або задавайте типову модель безпосередньо з Claude Code.

> ⚠️ **Неофіційний.** Це плагін від спільноти, не пов'язаний з Google чи Anthropic і не
> схвалений ними. «Antigravity», «Gemini», «Claude» та «Codex» належать їхнім
> відповідним власникам.

## Що ви отримуєте

| Команда | Що вона робить |
|---|---|
| `/agy:ask` | Поставити agy одноразове запитання (типово лише для читання) |
| `/agy:research` | Попросити agy дослідити та синтезувати відповідь |
| `/agy:rescue` | Делегувати завдання/виправлення — **agy може редагувати файли** |
| `/agy:review` | agy переглядає ваш локальний git diff (лише для читання) |
| `/agy:adversarial-review` | Безжальний змагальний перегляд вашого diff (лише для читання) |
| `/agy:model` | Показати або задати **типову** модель |
| `/agy:models` | Перелічити **всі** моделі, доступні вашому обліковому запису (Gemini / Claude / GPT-OSS) |
| `/agy:update` | Оновити agy CLI; оновлює список моделей |
| `/agy:setup` | Перевірити стан інтеграції |
| `/agy:install` | Встановити agy CLI (спершу запитує) |
| `/agy:status` `/agy:result` `/agy:cancel` | Керувати фоновими завданнями |

## Вимоги

- **Claude Code** (це плагін для нього)
- **Node.js 18+** (середовище виконання — Node; `node-pty` встановлюється автоматично під час першого запуску)
- **agy CLI** (Google Antigravity). Не маєте його? Запустіть `/agy:install` (він спершу
  запитує) або встановіть вручну з <https://antigravity.google>. Після встановлення запустіть
  `agy` один раз інтерактивно, щоб увійти.

Протестовано на **Windows** та **Linux** (x86_64). macOS має працювати (ті самі шляхи коду), але
не протестовано. Див. *Примітки щодо платформ* нижче для застереження стосовно Linux/SSH.

## Встановлення

> **Не бачите команди `/plugin`?** Ваш Claude Code застарий — `/plugin` потребує
> нещодавньої версії (2.1.143+). Спершу оновіть Claude Code (застосунок зі Store: оновлення
> через Microsoft Store / App store; CLI: `claude update`), потім перезапустіть його. Можливість
> користуватися новою *моделлю*, як-от Opus 4.8, **не** означає, що ваш застосунок оновлено —
> моделі надходять із сервера, а функція `/plugin` — із застосунку.

**Крок 1 — додайте маркетплейс і встановіть** (у Claude Code):

```bash
/plugin marketplace add alieuidsh/claude-agy-plugin
/plugin install agy@shuo-agy
```

**Крок 2 — зробіть, щоб команди з'явилися. ⚠️ Цей крок обов'язковий, і саме на ньому люди
застрягають.** Щойно встановлені команди **не** з'являються, доки ви не перезавантажите чи не перезапустите:

- Запустіть **`/reload-plugins`**, **і**
- якщо команди `/agy:*` досі не з'являються (або після *оновлення* плагіна),
  **повністю закрийте та знову відкрийте Claude Code** (закрийте вікно/застосунок повністю, а не
  лише вкладку). Самого лише перезавантаження інколи недостатньо для геть нових файлів команд.

**Крок 3 — перевірка стану:**

```bash
/agy:setup     # verifies agy + node-pty + auth; auto-installs node-pty on first run
```

Найперший виклик `/agy:*` займає ~15–20 с (одноразове встановлення node-pty + перше зчитування
списку моделей). Це нормально — після цього все кешується, і подальші виклики швидкі.

Перша команда, що задіює agy, може зайняти ~15–20 с (одноразове встановлення node-pty + зчитування
списку моделей, обидва кешуються згодом).

## Як це працює (і чому)

agy 1.0.x **видає вивід лише тоді, коли виявляє справжню консоль (TTY)** — звичайний
безголовий `spawn()` не дає нічого. Тож цей плагін задіює agy всередині **синтезованої
консолі (ConPTY) через `node-pty`**, читає її вивід, прибирає ANSI/BOM і повертає
відповідь. `node-pty` постачається з попередньо зібраними двійковими файлами для поширених
комбінацій Node/ОС і встановлюється автоматично під час першого використання (інструментарій C++ у
звичайному випадку не потрібен).

Список моделей зчитується наживо з інтерактивного меню `/model` у agy й кешується з прив'язкою до
відбитка двійкового файлу agy — він автоматично перечитується, коли agy оновлюється.

## Вибір моделі

agy **не має прапорця `--model` для CLI**, тож цей плагін обирає модель, коротко й безпечно
переписуючи `~/.gemini/antigravity-cli/settings.json`, а потім відновлюючи його. Це робиться
під блокуванням і є **стійким до збоїв** — ваші налаштування ніколи не лишаються пошкодженими,
навіть якщо запуск буде перервано посеред роботи (оригінал зберігається й відновлюється наступним запуском).

```bash
/agy:models                                  # see everything your account can run
/agy:model                                   # show the current default
/agy:model pro                               # set default to the strongest Gemini Pro
/agy:model flash                             # set default to Gemini Flash (fast, cheap)
/agy:model "Claude Opus 4.6 (Thinking)"      # default to a Claude model
/agy:ask --model flash  your question        # one-off override (doesn't change default)
```

- **Псевдоніми** (`pro`, `flash`, а також `pro-low`, `flash-medium`, …) працюють **лише для
  Gemini** і відстежують наживо список, тож `pro`/`flash` автоматично слідують за найновішим рівнем Gemini.
- Моделі **Claude / GPT-OSS** потребують **повної мітки** — скопіюйте її з `/agy:models`.
- Типова модель зберігається у `~/.agy-jobs/config.json` — миттєво, зберігається між сесіями,
  без перезапуску термінала. Прапорець `--model` для конкретного виклику завжди перемагає типову модель.
- Кожен запуск повідомляє про модель, **яка справді була використана** (зчитується з власного журналу
  agy, а не зі самозвіту моделі — моделі ненадійні в самоназиванні).

## Дозволи

- `ask` / `research` типово працюють **лише для читання**; додайте `--write`, щоб дозволити редагування.
- `rescue` типово **здатний записувати**; додайте `--read-only` лише для порад.
- `review` / `adversarial-review` **завжди лише для читання** — використовуйте `/agy:rescue`, щоб
  діяти за результатами.
- Запуски лише для читання передають agy прапорець `--sandbox` (обмеження термінала): agy все ще може
  читати й аналізувати файли, але системні/термінальні побічні дії заблоковано.

## Примітки щодо платформ

- **Windows / Linux** — повністю протестовано (перемикання моделей, зчитування, стійке до збоїв відновлення — усе працює).
- **Підводний камінь Linux + SSH**: agy зберігає ваш логін у keyring робочого стола, коли ви входите в
  графічній сесії, але перемикається на токени на основі файлів, коли виявляє SSH-сесію
  (`SSH_CONNECTION`). Ці два не поділяють стан, тож запуск плагіна **через «голе» SSH-з'єднання**
  може спричинити «Authentication required», навіть якщо ви ввійшли на робочому столі. Виправлення:
  увійдіть *усередині* SSH-сесії, **або** запускайте всередині сесії `tmux`/`screen`, яку було
  запущено з робочого стола (без `SSH_CONNECTION` у її середовищі) — тоді agy нормально читає логін
  робочого стола. Це поведінка agy CLI, а не вада плагіна.

## ⚠️ Конфіденційність — прочитайте це

agy надсилає ваші запити (а для `review` — ваш diff коду) на **сервери Google**. **Не**
використовуйте його з секретами, обліковими даними, приватними ключами чи конфіденційною /
неопублікованою роботою, якою ви не можете поділитися зі стороннім сервісом. Ставтеся до нього, як
до будь-якого іншого хмарного AI-сервісу.

## Усунення несправностей

- **Команду `/plugin` не знайдено** → ваш Claude Code застарий (нижче 2.1.143). Оновіть
  застосунок і перезапустіть його (див. [Встановлення](#install)). Можливість користуватися новою
  *моделлю* не означає, що застосунок оновлено.
- **Встановлено, але команди `/agy:*` не з'являються** → запустіть **`/reload-plugins`**; якщо
  вони досі не з'являються, **повністю закрийте та знову відкрийте** Claude Code. Нові файли команд
  потребують перезавантаження/перезапуску для завантаження.
- **`/agy:setup` повідомляє `agy binary: NOT FOUND`** → запустіть `/agy:install` або задайте
  змінну середовища `AGY_BIN` зі шляхом до виконуваного файлу agy.
- **`node-pty: UNAVAILABLE`** → одноразове автовстановлення не вдалося; переконайтеся, що Node.js + npm
  є в PATH і у вас є мережа, потім повторно запустіть `/agy:setup`.
- **Немає відповіді / помилка автентифікації** → запустіть `agy` один раз інтерактивно в терміналі, щоб увійти.
- **Список моделей виглядає застарілим після оновлення agy** → `/agy:models --refresh` або `/agy:update`.

Не повторюйте спроби циклічно після збою — усуньте першопричину (автентифікація, встановлення, мережа).
