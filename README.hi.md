# agy — Claude Code के लिए Google Antigravity

[English](README.md) · [繁體中文](README.zh-TW.md) · [简体中文](README.zh-CN.md) · [Español](README.es.md) · [हिन्दी](README.hi.md) · [العربية](README.ar.md) · [Português](README.pt-BR.md) · [Русский](README.ru.md) · [日本語](README.ja.md) · [Deutsch](README.de.md) · [Français](README.fr.md) · [한국어](README.ko.md) · [Italiano](README.it.md) · [Türkçe](README.tr.md) · [Tiếng Việt](README.vi.md) · [Bahasa Indonesia](README.id.md) · [Polski](README.pl.md) · [Nederlands](README.nl.md) · [ไทย](README.th.md) · [Українська](README.uk.md)

**agy CLI** (Google Antigravity) को Claude Code के अंदर एक दूसरे मॉडल के रूप में इस्तेमाल करें — यह `codex` plugin का `agy` समकक्ष है। सवाल पूछें, दूसरी राय लें, अपने diff की समीक्षा करवाएँ, या लिखने में सक्षम कार्य सौंपें — और यह सब Claude Code छोड़े बिना।

इसे दिलचस्प क्या बनाता है: **agy एक ही CLI और अकाउंट के पीछे Gemini, Claude, *और* GPT-OSS मॉडल चला सकता है**। यह plugin उसी को सामने लाता है — हर कॉल पर इनमें से कोई भी चुनें, या एक डिफ़ॉल्ट सेट करें, सीधे Claude Code से।

> ⚠️ **अनौपचारिक।** यह एक कम्युनिटी plugin है, जो Google या Anthropic से संबद्ध या उनके द्वारा अनुमोदित नहीं है। "Antigravity", "Gemini", "Claude", और "Codex" अपने-अपने स्वामियों के हैं।

---

## आपको क्या मिलता है

| कमांड | यह क्या करता है |
|---|---|
| `/agy:ask` | agy से एक बार में एक सवाल पूछें (डिफ़ॉल्ट रूप से read-only) |
| `/agy:research` | agy से किसी उत्तर पर शोध करने और उसे संश्लेषित करने को कहें |
| `/agy:rescue` | कोई कार्य/फ़िक्स सौंपें — **agy फ़ाइलें संपादित कर सकता है** |
| `/agy:review` | agy आपके स्थानीय git diff की समीक्षा करता है (read-only) |
| `/agy:adversarial-review` | आपके diff की निर्मम विरोधात्मक समीक्षा (read-only) |
| `/agy:model` | **डिफ़ॉल्ट** मॉडल दिखाएँ या सेट करें |
| `/agy:models` | वे **सभी** मॉडल सूचीबद्ध करें जो आपका अकाउंट इस्तेमाल कर सकता है (Gemini / Claude / GPT-OSS) |
| `/agy:update` | agy CLI अपडेट करें; मॉडल सूची ताज़ा करता है |
| `/agy:setup` | इंटीग्रेशन का स्वास्थ्य-जाँच करें |
| `/agy:install` | agy CLI इंस्टॉल करें (पहले पूछता है) |
| `/agy:status` `/agy:result` `/agy:cancel` | बैकग्राउंड जॉब्स प्रबंधित करें |

## आवश्यकताएँ

- **Claude Code** (यह उसी के लिए एक plugin है)
- **Node.js 18+** (रनटाइम Node है; `node-pty` पहली बार चलाने पर स्वतः इंस्टॉल हो जाता है)
- **agy CLI** (Google Antigravity)। आपके पास नहीं है? `/agy:install` चलाएँ (यह पहले पूछता है), या <https://antigravity.google> से मैन्युअल रूप से इंस्टॉल करें। इंस्टॉल करने के बाद, साइन इन करने के लिए एक बार इंटरैक्टिव रूप से `agy` चलाएँ।

**Windows** और **Linux** (x86_64) पर परखा गया। macOS काम करना चाहिए (वही कोड पाथ) लेकिन इसे परखा नहीं गया है। Linux/SSH चेतावनी के लिए नीचे *प्लेटफ़ॉर्म नोट्स* देखें।

## इंस्टॉल

> **`/plugin` कमांड नहीं दिख रहा?** आपका Claude Code बहुत पुराना है — `/plugin` के लिए एक हाल का संस्करण (2.1.143+) चाहिए। पहले Claude Code अपडेट करें (Store app: Microsoft Store / App store के ज़रिए अपडेट करें; CLI: `claude update`), फिर इसे रीस्टार्ट करें। Opus 4.8 जैसे किसी नए *मॉडल* का इस्तेमाल कर पाने का मतलब यह **नहीं** है कि आपकी ऐप अप टू डेट है — मॉडल सर्वर से आते हैं, `/plugin` फ़ीचर ऐप से आता है।

**चरण 1 — मार्केटप्लेस जोड़ें और इंस्टॉल करें** (Claude Code में):

```bash
/plugin marketplace add alieuidsh/claude-agy-plugin
/plugin install agy@shuo-agy
```

**चरण 2 — कमांड्स को सामने लाएँ। ⚠️ यह चरण आवश्यक है, और यहीं लोग अटक जाते हैं।** नए इंस्टॉल किए गए कमांड्स तब तक **नहीं** दिखते जब तक आप रीलोड या रीस्टार्ट न करें:

- **`/reload-plugins`** चलाएँ, **और**
- अगर `/agy:*` कमांड्स फिर भी न दिखें (या किसी plugin *अपडेट* के बाद), **Claude Code को पूरी तरह बंद करके फिर से खोलें** (विंडो/ऐप को पूरी तरह बंद करें, सिर्फ़ टैब नहीं)। नई बनी कमांड फ़ाइलों के लिए कभी-कभी अकेला रीलोड पर्याप्त नहीं होता।

**चरण 3 — स्वास्थ्य-जाँच:**

```bash
/agy:setup     # verifies agy + node-pty + auth; auto-installs node-pty on first run
```

पहली ही `/agy:*` कॉल में ~15–20 सेकंड लगते हैं (एक बार का node-pty इंस्टॉल + पहली मॉडल-सूची स्क्रैप)। यह सामान्य है — उसके बाद यह कैश हो जाता है, बाद की कॉल्स तेज़ होती हैं।

agy को चलाने वाली पहली कमांड में ~15–20 सेकंड लग सकते हैं (एक बार का node-pty इंस्टॉल + एक मॉडल सूची स्क्रैप, दोनों बाद में कैश हो जाते हैं)।

## यह कैसे काम करता है (और क्यों)

agy 1.0.x **तभी आउटपुट देता है जब उसे एक असली कंसोल (TTY) मिले** — एक सादा हेडलेस `spawn()` कुछ नहीं देता। इसलिए यह plugin agy को एक **संश्लेषित कंसोल (ConPTY) के अंदर `node-pty` के ज़रिए** चलाता है, उसका आउटपुट पढ़ता है, ANSI/BOM हटाता है, और उत्तर लौटाता है। `node-pty` आम Node/OS संयोजनों के लिए पहले से बने बाइनरी के साथ आता है और पहली बार इस्तेमाल पर स्वतः इंस्टॉल हो जाता है (सामान्य स्थिति में किसी C++ टूलचेन की ज़रूरत नहीं)।

मॉडल सूची agy के इंटरैक्टिव `/model` मेनू से लाइव स्क्रैप की जाती है और कैश की जाती है, जो agy बाइनरी के फ़िंगरप्रिंट पर आधारित होती है — agy अपडेट होने पर यह स्वतः फिर से स्क्रैप हो जाती है।

## मॉडल चुनना

agy में **कोई `--model` CLI फ़्लैग नहीं है**, इसलिए यह plugin संक्षेप में और सुरक्षित रूप से `~/.gemini/antigravity-cli/settings.json` को फिर से लिखकर मॉडल चुनता है, फिर उसे पुनर्स्थापित कर देता है। यह एक लॉक के अंतर्गत किया जाता है और **क्रैश-सेफ़** है — आपकी सेटिंग्स कभी भ्रष्ट नहीं छूटतीं, भले ही कोई रन बीच में ही समाप्त कर दिया जाए (मूल को सहेज लिया जाता है और अगले रन द्वारा पुनः प्राप्त कर लिया जाता है)।

```bash
/agy:models                                  # see everything your account can run
/agy:model                                   # show the current default
/agy:model pro                               # set default to the strongest Gemini Pro
/agy:model flash                             # set default to Gemini Flash (fast, cheap)
/agy:model "Claude Opus 4.6 (Thinking)"      # default to a Claude model
/agy:ask --model flash  your question        # one-off override (doesn't change default)
```

- **उपनाम (aliases)** (`pro`, `flash`, और साथ ही `pro-low`, `flash-medium`, …) **केवल Gemini के लिए** हैं और लाइव सूची का अनुसरण करते हैं, इसलिए `pro`/`flash` स्वतः नवीनतम Gemini टियर का पालन करते हैं।
- **Claude / GPT-OSS** मॉडलों के लिए **पूरा लेबल** चाहिए — इसे `/agy:models` से कॉपी करें।
- डिफ़ॉल्ट `~/.agy-jobs/config.json` में सहेजा जाता है — तत्काल, सत्रों के बीच बना रहता है, किसी टर्मिनल रीस्टार्ट की ज़रूरत नहीं। प्रति-कॉल `--model` हमेशा डिफ़ॉल्ट पर भारी पड़ता है।
- हर रन उस मॉडल की रिपोर्ट देता है जो **वास्तव में इस्तेमाल हुआ** (agy के अपने लॉग से पढ़ा गया, मॉडल की स्वयं-रिपोर्ट से नहीं — मॉडल अपना नाम बताने में अविश्वसनीय होते हैं)।

## अनुमतियाँ

- `ask` / `research` डिफ़ॉल्ट रूप से **read-only** हैं; संपादन की अनुमति देने के लिए `--write` जोड़ें।
- `rescue` डिफ़ॉल्ट रूप से **लिखने में सक्षम** है; केवल सलाह के लिए `--read-only` जोड़ें।
- `review` / `adversarial-review` **हमेशा read-only** हैं — निष्कर्षों पर कार्रवाई के लिए `/agy:rescue` का इस्तेमाल करें।
- Read-only रन agy का `--sandbox` (टर्मिनल प्रतिबंध) पास करते हैं: agy फिर भी फ़ाइलें पढ़ और विश्लेषित कर सकता है, लेकिन सिस्टम/टर्मिनल साइड-इफ़ेक्ट अवरुद्ध रहते हैं।

## प्लेटफ़ॉर्म नोट्स

- **Windows / Linux** — पूरी तरह परखा गया (मॉडल स्विचिंग, स्क्रैप, क्रैश-सेफ़ पुनर्स्थापन सब काम करते हैं)।
- **Linux + SSH की पेंच**: जब आप किसी ग्राफ़िकल सत्र में साइन इन करते हैं तो agy अपना लॉगिन डेस्कटॉप keyring में संग्रहीत करता है, लेकिन जब उसे कोई SSH सत्र (`SSH_CONNECTION`) मिलता है तो वह फ़ाइल-आधारित टोकन पर स्विच कर जाता है। दोनों स्थिति साझा नहीं करते, इसलिए plugin को **एक नंगे SSH कनेक्शन पर** चलाने से "Authentication required" आ सकता है, भले ही आप डेस्कटॉप पर लॉग इन हों। समाधान: SSH सत्र के *भीतर* साइन इन करें, **या** एक ऐसे `tmux`/`screen` सत्र के अंदर चलाएँ जो डेस्कटॉप से शुरू किया गया हो (जिसके परिवेश में `SSH_CONNECTION` न हो) — तब agy डेस्कटॉप लॉगिन सामान्य रूप से पढ़ लेता है। यह agy CLI का व्यवहार है, कोई plugin बग नहीं।

## ⚠️ गोपनीयता — इसे ज़रूर पढ़ें

agy आपके प्रॉम्प्ट (और, `review` के लिए, आपका कोड diff) **Google के सर्वरों** को भेजता है। इसे रहस्यों, क्रेडेंशियल्स, निजी कुंजियों, या ऐसे गोपनीय / अप्रकाशित काम पर इस्तेमाल **न** करें जिसे आप किसी तीसरे पक्ष के साथ साझा नहीं कर सकते। इसे किसी भी अन्य क्लाउड AI सेवा की तरह ही समझें।

## समस्या-निवारण

- **`/plugin` कमांड नहीं मिला** → आपका Claude Code बहुत पुराना है (2.1.143 से नीचे)। ऐप अपडेट करें और रीस्टार्ट करें ([इंस्टॉल](#install) देखें)। किसी नए *मॉडल* का इस्तेमाल कर पाने का मतलब यह नहीं कि ऐप अप टू डेट है।
- **इंस्टॉल हो गया, पर `/agy:*` कमांड्स नहीं दिख रहे** → **`/reload-plugins`** चलाएँ; अगर वे फिर भी न दिखें, तो Claude Code को **पूरी तरह बंद करके फिर से खोलें**। नई कमांड फ़ाइलों को लोड होने के लिए रीलोड/रीस्टार्ट चाहिए।
- **`/agy:setup` कहता है `agy binary: NOT FOUND`** → `/agy:install` चलाएँ, या `AGY_BIN` एनवायरनमेंट वेरिएबल को agy एक्ज़ीक्यूटेबल के पाथ पर सेट करें।
- **`node-pty: UNAVAILABLE`** → एक बार का स्वतः-इंस्टॉल विफल हो गया; सुनिश्चित करें कि Node.js + npm PATH पर हैं और आपके पास नेटवर्क है, फिर `/agy:setup` फिर से चलाएँ।
- **कोई उत्तर नहीं / auth त्रुटि** → साइन इन करने के लिए एक बार किसी टर्मिनल में इंटरैक्टिव रूप से `agy` चलाएँ।
- **agy अपडेट के बाद मॉडल सूची पुरानी लगती है** → `/agy:models --refresh` या `/agy:update`।

विफलता पर लूप में पुनः-प्रयास न करें — अंतर्निहित कारण ठीक करें (auth, इंस्टॉल, नेटवर्क)।
