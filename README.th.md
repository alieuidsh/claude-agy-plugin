# agy — Google Antigravity สำหรับ Claude Code

[English](README.md) · [繁體中文](README.zh-TW.md) · [简体中文](README.zh-CN.md) · [Español](README.es.md) · [हिन्दी](README.hi.md) · [العربية](README.ar.md) · [Português](README.pt-BR.md) · [Русский](README.ru.md) · [日本語](README.ja.md) · [Deutsch](README.de.md) · [Français](README.fr.md) · [한국어](README.ko.md) · [Italiano](README.it.md) · [Türkçe](README.tr.md) · [Tiếng Việt](README.vi.md) · [Bahasa Indonesia](README.id.md) · [Polski](README.pl.md) · [Nederlands](README.nl.md) · [ไทย](README.th.md) · [Українська](README.uk.md)

ใช้ **agy CLI** (Google Antigravity) เป็นโมเดลที่สองภายใน Claude Code — เป็นคู่หู
`agy` ของปลั๊กอิน `codex` ถามคำถาม ขอความเห็นที่สอง รีวิว
diff ของคุณ หรือมอบหมายงานที่เขียนไฟล์ได้ ทั้งหมดนี้โดยไม่ต้องออกจาก Claude Code

สิ่งที่ทำให้มันน่าสนใจ: **agy สามารถรันโมเดล Gemini, Claude, *และ* GPT-OSS** ได้ผ่าน
CLI และบัญชีเดียว ปลั๊กอินนี้นำความสามารถนั้นออกมาให้เห็น — เลือกโมเดลใดก็ได้ในแต่ละการเรียกใช้ หรือตั้ง
ค่าเริ่มต้น ได้โดยตรงจาก Claude Code

> ⚠️ **ไม่เป็นทางการ** นี่คือปลั๊กอินจากชุมชน ไม่ได้มีส่วนเกี่ยวข้องหรือได้รับการรับรองจาก
> Google หรือ Anthropic "Antigravity", "Gemini", "Claude" และ "Codex" เป็นของ
> เจ้าของตามลำดับ

## สิ่งที่คุณจะได้รับ

| คำสั่ง | หน้าที่ |
|---|---|
| `/agy:ask` | ถาม agy แบบครั้งเดียว (อ่านอย่างเดียวโดยค่าเริ่มต้น) |
| `/agy:research` | ให้ agy ค้นคว้าและสังเคราะห์คำตอบ |
| `/agy:rescue` | มอบหมายงาน/การแก้ไข — **agy อาจแก้ไขไฟล์ได้** |
| `/agy:review` | agy รีวิว git diff ในเครื่องของคุณ (อ่านอย่างเดียว) |
| `/agy:adversarial-review` | รีวิวแบบโต้แย้งอย่างเข้มงวดต่อ diff ของคุณ (อ่านอย่างเดียว) |
| `/agy:model` | แสดงหรือตั้งค่าโมเดล **เริ่มต้น** |
| `/agy:models` | แสดงรายการโมเดล **ทั้งหมด** ที่บัญชีของคุณใช้ได้ (Gemini / Claude / GPT-OSS) |
| `/agy:update` | อัปเดต agy CLI; รีเฟรชรายการโมเดล |
| `/agy:setup` | ตรวจสุขภาพการเชื่อมต่อ |
| `/agy:install` | ติดตั้ง agy CLI (ถามก่อน) |
| `/agy:status` `/agy:result` `/agy:cancel` | จัดการงานเบื้องหลัง |

## ความต้องการของระบบ

- **Claude Code** (นี่คือปลั๊กอินสำหรับมัน)
- **Node.js 18+** (รันไทม์คือ Node; `node-pty` จะถูกติดตั้งอัตโนมัติในการรันครั้งแรก)
- **agy CLI** (Google Antigravity) ยังไม่มีใช่ไหม? รัน `/agy:install` (มันจะถาม
  ก่อน) หรือติดตั้งด้วยตนเองจาก <https://antigravity.google> หลังติดตั้งแล้ว รัน
  `agy` หนึ่งครั้งแบบโต้ตอบเพื่อลงชื่อเข้าใช้

ทดสอบบน **Windows** และ **Linux** (x86_64) แล้ว macOS น่าจะทำงานได้ (เส้นทางโค้ดเดียวกัน) แต่ยัง
ไม่ได้ทดสอบ ดู *หมายเหตุเกี่ยวกับแพลตฟอร์ม* ด้านล่างสำหรับข้อควรระวังเรื่อง Linux/SSH

## การติดตั้ง

> **ไม่เห็นคำสั่ง `/plugin`?** Claude Code ของคุณเก่าเกินไป — `/plugin` ต้องใช้
> เวอร์ชันใหม่ (2.1.143+) อัปเดต Claude Code ก่อน (แอป Store: อัปเดตผ่าน Microsoft
> Store / App store; CLI: `claude update`) แล้วรีสตาร์ท การที่สามารถใช้
> *โมเดล* ใหม่อย่าง Opus 4.8 ได้ **ไม่ได้** หมายความว่าแอปของคุณเป็นเวอร์ชันล่าสุด — โมเดลมาจาก
> เซิร์ฟเวอร์ ส่วนฟีเจอร์ `/plugin` มาจากแอป

**ขั้นที่ 1 — เพิ่ม marketplace และติดตั้ง** (ใน Claude Code):

```bash
/plugin marketplace add alieuidsh/claude-agy-plugin
/plugin install agy@shuo-agy
```

**ขั้นที่ 2 — ทำให้คำสั่งปรากฏ ⚠️ ขั้นตอนนี้จำเป็น และเป็นจุดที่คนมัก
ติดขัด** คำสั่งที่เพิ่งติดตั้งจะ **ไม่** ปรากฏจนกว่าคุณจะรีโหลดหรือรีสตาร์ท:

- รัน **`/reload-plugins`** **และ**
- หากคำสั่ง `/agy:*` ยังไม่ปรากฏ (หรือหลังจาก *อัปเดต* ปลั๊กอิน)
  ให้ **ปิดและเปิด Claude Code ใหม่ทั้งหมด** (ปิดหน้าต่าง/แอปให้สนิท ไม่ใช่แค่
  แท็บ) บางครั้งการรีโหลดอย่างเดียวไม่เพียงพอสำหรับไฟล์คำสั่งใหม่เอี่ยม

**ขั้นที่ 3 — ตรวจสุขภาพ:**

```bash
/agy:setup     # verifies agy + node-pty + auth; auto-installs node-pty on first run
```

การเรียก `/agy:*` ครั้งแรกสุดจะใช้เวลาประมาณ 15–20 วินาที (ติดตั้ง node-pty ครั้งเดียว + ดึงรายการ
โมเดลครั้งแรก) นั่นเป็นเรื่องปกติ — หลังจากนั้นจะถูกแคชไว้ การเรียกครั้งต่อ ๆ ไปจะเร็ว

คำสั่งแรกที่ขับเคลื่อน agy อาจใช้เวลาประมาณ 15–20 วินาที (ติดตั้ง node-pty ครั้งเดียว + ดึงรายการ
โมเดล ทั้งคู่ถูกแคชไว้หลังจากนั้น)

## วิธีการทำงาน (และเหตุผล)

agy 1.0.x **จะให้ผลลัพธ์ก็ต่อเมื่อมันตรวจพบคอนโซลจริง (TTY)** เท่านั้น — การ
`spawn()` แบบ headless ธรรมดาจะไม่ได้อะไรเลย ดังนั้นปลั๊กอินนี้จึงขับเคลื่อน agy ภายใน **คอนโซล
ที่สังเคราะห์ขึ้น (ConPTY) ผ่าน `node-pty`** อ่านผลลัพธ์ของมัน ลบ ANSI/BOM ออก แล้วส่งคืน
คำตอบ `node-pty` มาพร้อมไบนารีที่คอมไพล์ไว้ล่วงหน้าสำหรับชุดค่าผสม Node/OS ทั่วไป และถูกติดตั้ง
อัตโนมัติในการใช้งานครั้งแรก (โดยทั่วไปไม่จำเป็นต้องมี C++ toolchain)

รายการโมเดลถูกดึงสด ๆ จากเมนู `/model` แบบโต้ตอบของ agy และแคชไว้ โดยอ้างอิงตาม
ลายนิ้วมือของไบนารี agy — มันจะดึงใหม่อัตโนมัติเมื่อ agy อัปเดต

## การเลือกโมเดล

agy **ไม่มีแฟล็ก `--model` ใน CLI** ดังนั้นปลั๊กอินนี้จึงเลือกโมเดลโดยเขียนทับ
`~/.gemini/antigravity-cli/settings.json` ชั่วครู่อย่างปลอดภัย แล้วคืนค่าเดิม ทั้งหมดนี้ทำ
ภายใต้การล็อก และ **ปลอดภัยต่อการแครช** — การตั้งค่าของคุณจะไม่มีวันเสียหาย แม้ว่าการ
รันจะถูกฆ่ากลางคัน (ต้นฉบับจะถูกเก็บไว้และกู้คืนโดยการรันครั้งถัดไป)

```bash
/agy:models                                  # see everything your account can run
/agy:model                                   # show the current default
/agy:model pro                               # set default to the strongest Gemini Pro
/agy:model flash                             # set default to Gemini Flash (fast, cheap)
/agy:model "Claude Opus 4.6 (Thinking)"      # default to a Claude model
/agy:ask --model flash  your question        # one-off override (doesn't change default)
```

- **ชื่อย่อ** (`pro`, `flash` รวมถึง `pro-low`, `flash-medium`, …) เป็น **ของ Gemini เท่านั้น** และ
  ติดตามรายการสด ดังนั้น `pro`/`flash` จะตามระดับ Gemini ใหม่ล่าสุดโดยอัตโนมัติ
- โมเดล **Claude / GPT-OSS** ต้องใช้ **ป้ายชื่อเต็ม** — คัดลอกมาจาก `/agy:models`
- ค่าเริ่มต้นจะถูกบันทึกไว้ที่ `~/.agy-jobs/config.json` — ทันที คงอยู่ข้ามเซสชัน
  ไม่ต้องรีสตาร์ทเทอร์มินัล `--model` ต่อการเรียกหนึ่งครั้งจะชนะค่าเริ่มต้นเสมอ
- การรันทุกครั้งจะรายงานโมเดลที่ **ใช้จริง** (อ่านจากบันทึกของ agy เอง ไม่ใช่จากที่โมเดล
  รายงานตัวเอง — โมเดลมักเชื่อถือไม่ได้ในการบอกชื่อตัวเอง)

## สิทธิ์การเข้าถึง

- `ask` / `research` เป็น **อ่านอย่างเดียว** โดยค่าเริ่มต้น; เพิ่ม `--write` เพื่ออนุญาตให้แก้ไขได้
- `rescue` **เขียนไฟล์ได้** โดยค่าเริ่มต้น; เพิ่ม `--read-only` หากต้องการเพียงคำแนะนำ
- `review` / `adversarial-review` เป็น **อ่านอย่างเดียวเสมอ** — ใช้ `/agy:rescue` เพื่อดำเนินการกับ
  สิ่งที่ค้นพบ
- การรันแบบอ่านอย่างเดียวจะส่ง `--sandbox` ของ agy (ข้อจำกัดของเทอร์มินัล): agy ยังอ่านและ
  วิเคราะห์ไฟล์ได้ แต่ผลข้างเคียงต่อระบบ/เทอร์มินัลจะถูกบล็อก

## หมายเหตุเกี่ยวกับแพลตฟอร์ม

- **Windows / Linux** — ทดสอบครบถ้วน (การสลับโมเดล การดึงข้อมูล การคืนค่าแบบปลอดภัยต่อการแครช ทำงานได้ทั้งหมด)
- **ข้อควรระวังเรื่อง Linux + SSH**: agy เก็บข้อมูลการเข้าสู่ระบบไว้ใน keyring ของเดสก์ท็อปเมื่อคุณลงชื่อเข้าใช้ที่
  เซสชันแบบกราฟิก แต่จะเปลี่ยนไปใช้โทเค็นแบบไฟล์เมื่อมันตรวจพบเซสชัน SSH
  (`SSH_CONNECTION`) ทั้งสองไม่ได้แชร์สถานะกัน ดังนั้นการรันปลั๊กอิน **ผ่านการเชื่อมต่อ SSH
  เปล่า ๆ** อาจพบ "Authentication required" แม้ว่าคุณจะลงชื่อเข้าใช้บน
  เดสก์ท็อปแล้ว วิธีแก้: ลงชื่อเข้าใช้ *ภายใน* เซสชัน SSH **หรือ** รันภายในเซสชัน `tmux`/`screen`
  ที่เริ่มต้นจากเดสก์ท็อป (ไม่มี `SSH_CONNECTION` ในสภาพแวดล้อมของมัน) —
  จากนั้น agy จะอ่านข้อมูลการเข้าสู่ระบบของเดสก์ท็อปได้ตามปกติ นี่เป็นพฤติกรรมของ agy CLI ไม่ใช่บั๊กของปลั๊กอิน

## ⚠️ ความเป็นส่วนตัว — อ่านสิ่งนี้

agy ส่งพรอมต์ของคุณ (และสำหรับ `review` คือ code diff ของคุณ) ไปยัง **เซิร์ฟเวอร์ของ Google**
**อย่า** ใช้มันกับความลับ ข้อมูลรับรอง คีย์ส่วนตัว หรืองานที่เป็นความลับ / ยังไม่เผยแพร่
ที่คุณไม่สามารถแบ่งปันกับบุคคลที่สามได้ ปฏิบัติต่อมันเหมือนบริการ AI บนคลาวด์อื่น ๆ

## การแก้ไขปัญหา

- **ไม่พบคำสั่ง `/plugin`** → Claude Code ของคุณเก่าเกินไป (ต่ำกว่า 2.1.143) อัปเดต
  แอปและรีสตาร์ท (ดู [การติดตั้ง](#install)) การที่สามารถใช้ *โมเดล* ใหม่ได้
  ไม่ได้หมายความว่าแอปเป็นเวอร์ชันล่าสุด
- **ติดตั้งแล้ว แต่คำสั่ง `/agy:*` ไม่ปรากฏ** → รัน **`/reload-plugins`**; หาก
  ยังไม่ปรากฏ ให้ **ปิดและเปิด Claude Code ใหม่ทั้งหมด** ไฟล์คำสั่งใหม่ต้องการ
  การรีโหลด/รีสตาร์ทเพื่อโหลด
- **`/agy:setup` แจ้งว่า `agy binary: NOT FOUND`** → รัน `/agy:install` หรือตั้งค่า
  ตัวแปรสภาพแวดล้อม `AGY_BIN` ให้เป็นเส้นทางของไฟล์ปฏิบัติการ agy
- **`node-pty: UNAVAILABLE`** → การติดตั้งอัตโนมัติครั้งเดียวล้มเหลว; ตรวจสอบว่า Node.js + npm
  อยู่ใน PATH และคุณมีเครือข่าย แล้วรัน `/agy:setup` ใหม่
- **ไม่มีคำตอบ / ข้อผิดพลาดการยืนยันตัวตน** → รัน `agy` หนึ่งครั้งแบบโต้ตอบในเทอร์มินัลเพื่อลงชื่อเข้าใช้
- **รายการโมเดลดูเก่าหลังจากอัปเดต agy** → `/agy:models --refresh` หรือ `/agy:update`

อย่าวนลูปลองใหม่เมื่อล้มเหลว — แก้ที่ต้นเหตุ (การยืนยันตัวตน การติดตั้ง เครือข่าย)
