# claude-agy-plugin — Google Antigravity (Gemini) для Claude Code

**Языки:** [English](README.md) · [繁體中文](README.zh-TW.md) · [简体中文](README.zh-CN.md) · [日本語](README.ja.md) · **Русский** · [Deutsch](README.de.md) · [Français](README.fr.md) · [Español](README.es.md) · [Italiano](README.it.md)

Плагин для Claude Code, позволяющий Claude вызывать локальный CLI **`agy`**
(Google Antigravity на базе Gemini) для ревью кода, делегирования задач и получения
второго мнения — **аналог официального плагина `codex`**.

Кроссплатформенность: **Windows, Linux, macOS** (чистый Node-компаньон, без shell-обёрток).

## Команды

| Команда | Назначение | Права по умолчанию | Аналог в codex |
|---|---|---|---|
| `/agy:ask` | Разовый вопрос к Gemini | только чтение | — |
| `/agy:rescue` | Делегировать задачу/исправление (agy может **править файлы**) | **запись** | `/codex:rescue` |
| `/agy:research` | Исследовательский вопрос | только чтение | — |
| `/agy:review` | Ревью локального git diff | только чтение | `/codex:review` |
| `/agy:adversarial-review` | Состязательное ревью diff | только чтение | `/codex:adversarial-review` |
| `/agy:setup` | Проверка: установлен и авторизован? | — | `/codex:setup` |
| `/agy:status` | Список недавних задач agy | — | `/codex:status` |
| `/agy:result` | Показать вывод задачи | — | `/codex:result` |
| `/agy:cancel` | Отменить выполняющиеся задачи agy | — | `/codex:cancel` |

Можно просто сказать «спроси agy / дай второе мнение / спроси Gemini» — встроенный
skill сработает автоматически.

### Права: безопасное умолчание, выбор за вами

Как и в codex, у каждой команды есть **безопасное умолчание**, которое можно переопределить:

- `--write` — разрешить agy править файлы (переопределяет режим только чтения)
- `--read-only` — запретить правки, только советы (переопределяет режим записи)

То есть `/agy:ask --write` разрешает правки, а `/agy:rescue --read-only` оставляет
только советы без изменения файлов.

## Требования

1. **Node.js** (v18+). Проверка: `node --version`.
2. **CLI `agy`** (Google Antigravity), установлен и один раз выполнен вход.
   - **Нет CLI?** Выполните **`/agy:install`** — плагин обнаружит отсутствие и
     установит его (после запроса). Или вручную:
     - Windows: `irm https://antigravity.google/cli/install.ps1 | iex`
     - macOS/Linux: `curl -fsSL https://antigravity.google/cli/install.sh | bash`
   - Если установлен в другом месте — задайте переменную `AGY_BIN` с полным путём.
   - После установки запустите `agy` интерактивно один раз для входа в аккаунт Google.
   - Windows по умолчанию: `%LOCALAPPDATA%\agy\bin\agy.exe`
   - Linux/macOS: `~/.agy/bin/agy` или в `PATH`
   - Если установлен в другом месте — задайте переменную окружения `AGY_BIN`
     с полным путём.
   - Аутентификация выполняется незаметно через системный keyring. Если вызовы
     завершаются по таймауту, запустите `agy` интерактивно один раз для входа.

После установки проверьте всё командой `/agy:setup`.

## Установка

```
/plugin marketplace add alieuidsh/claude-agy-plugin
/plugin install agy@suho-agy
```

Затем перезапустите Claude Code и выполните `/agy:setup`.

## Обновление

Плагин распространяется через GitHub, поэтому каждый может получить свежие исправления:

```
/plugin update agy
```

Затем перезапустите Claude Code. (Эквивалент в CLI: `claude plugin update agy`.)
Если какая-то команда сообщает, что формат транскрипта agy не распознан — значит
изменился сам CLI `agy`; выполните обновление выше, чтобы получить исправленный плагин.

## Как это работает

`scripts/agy-companion.mjs` запускает `agy --print` с правильным порядком флагов,
ждёт с таймаутом и извлекает ответ Gemini из транскрипта, который agy пишет на диск
(`agy --print` пишет в TTY, а не в stdout). Каждый запуск получает уникальный nonce
задачи, чтобы читать правильный транскрипт даже при параллельных запусках, и все
сегменты ответа объединяются (agy дробит длинные ответы). Промпт пользователя
передаётся через **stdin**, поэтому пользовательский текст не попадает в командную
строку shell (нет инъекций). Задачи отслеживаются вместе с PID в `~/.agy-jobs`, чтобы
`/agy:status`, `/agy:result` и `/agy:cancel` работали между фоновыми запусками и
отменяли только нужную задачу.

## Безопасность и приватность (обязательно прочитайте)

- **Ваш код/промпты отправляются в Google (Gemini).** Учитывайте конфиденциальные,
  медицинские или неопубликованные данные — те же соображения, что и для любой
  облачной модели.
- **Команды с правом записи правят файлы автономно.** `/agy:rescue` может изменить
  файлы без подтверждения. Сначала сделайте `git commit`, чтобы потом проверить/откатить
  через `git diff`.
- agy — автономный агент: даже в режиме только чтения он может выполнять shell-команды
  для исследования. Режим только чтения блокирует *правку файлов*, а не всю активность.

## Примечания / ограничения

- agy использует один исполняемый файл для всех задач; `/agy:cancel` без id завершит
  все выполняющиеся задачи (с id — только дерево процессов этой задачи).
- Модель backend зависит от настроек agy (по умолчанию Gemini),
  в `~/.gemini/antigravity-cli/settings.json`.
