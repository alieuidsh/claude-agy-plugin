Используйте **agy CLI** (Google Antigravity) как вторую модель внутри Claude Code — это
аналог `agy` для плагина `codex`. Задавайте вопросы, получайте второе мнение, проверяйте
свой diff или делегируйте задачи с правом записи — и всё это не покидая Claude Code.

Что в этом интересного: **agy может запускать модели Gemini, Claude *и* GPT-OSS** за
единым CLI и одной учётной записью. Этот плагин даёт к этому доступ — выбирайте любую из
них для каждого вызова или задайте модель по умолчанию прямо из Claude Code.

> ⚠️ **Неофициально.** Это плагин от сообщества, не связанный с Google или Anthropic и не
> одобренный ими. «Antigravity», «Gemini», «Claude» и «Codex» принадлежат их
> соответствующим владельцам.

---

## Что вы получаете

| Команда | Что она делает |
|---|---|
| `/agy:ask` | Задать agy разовый вопрос (по умолчанию только чтение) |
| `/agy:research` | Попросить agy провести исследование и синтезировать ответ |
| `/agy:rescue` | Делегировать задачу/исправление — **agy может редактировать файлы** |
| `/agy:review` | agy проверяет ваш локальный git diff (только чтение) |
| `/agy:adversarial-review` | Беспощадная состязательная проверка вашего diff (только чтение) |
| `/agy:model` | Показать или задать модель **по умолчанию** |
| `/agy:models` | Показать **все** модели, доступные вашей учётной записи (Gemini / Claude / GPT-OSS) |
| `/agy:update` | Обновить agy CLI; обновляет список моделей |
| `/agy:setup` | Проверить работоспособность интеграции |
| `/agy:install` | Установить agy CLI (сначала спрашивает) |
| `/agy:status` `/agy:result` `/agy:cancel` | Управлять фоновыми задачами |

## Требования

- **Claude Code** (это плагин для него)
- **Node.js 18+** (среда выполнения — Node; `node-pty` устанавливается автоматически при первом запуске)
- **agy CLI** (Google Antigravity). Его нет? Запустите `/agy:install` (он сначала
  спрашивает) или установите вручную с <https://antigravity.google>. После установки один
  раз запустите `agy` в интерактивном режиме, чтобы войти.

Протестировано на **Windows** и **Linux** (x86_64). На macOS должно работать (те же пути
в коде), но не протестировано. См. раздел *Замечания по платформам* ниже о нюансе с Linux/SSH.

## Установка

> **Не видите команды `/plugin`?** Ваш Claude Code слишком старый — `/plugin` требует
> свежей версии (2.1.143+). Сначала обновите Claude Code (приложение Store: обновление
> через Microsoft Store / App store; CLI: `claude update`), затем перезапустите его.
> Возможность использовать новую *модель* вроде Opus 4.8 **не** означает, что ваше
> приложение обновлено — модели приходят с сервера, а функция `/plugin` — от приложения.

**Шаг 1 — добавьте маркетплейс и установите** (в Claude Code):

```bash
/plugin marketplace add alieuidsh/claude-agy-plugin
/plugin install agy@shuo-agy
```

**Шаг 2 — сделайте так, чтобы команды появились. ⚠️ Этот шаг обязателен, и именно на нём
люди застревают.** Только что установленные команды **не** появятся, пока вы не
перезагрузите или не перезапустите:

- Запустите **`/reload-plugins`**, **и**
- если команды `/agy:*` всё ещё не появляются (или после *обновления* плагина),
  **полностью закройте и снова откройте Claude Code** (закройте окно/приложение целиком, а
  не только вкладку). Одной перезагрузки иногда недостаточно для совсем новых файлов команд.

**Шаг 3 — проверка работоспособности:**

```bash
/agy:setup     # verifies agy + node-pty + auth; auto-installs node-pty on first run
```

Самый первый вызов `/agy:*` занимает ~15–20 с (разовая установка node-pty + первое
считывание списка моделей). Это нормально — после этого результат кэшируется, и
последующие вызовы выполняются быстро.

Первая команда, запускающая agy, может занять ~15–20 с (разовая установка node-pty + одно
считывание списка моделей, оба кэшируются после).

## Как это работает (и почему)

agy 1.0.x **выдаёт вывод только тогда, когда обнаруживает настоящую консоль (TTY)** —
обычный безголовый `spawn()` не возвращает ничего. Поэтому этот плагин запускает agy
внутри **синтезированной консоли (ConPTY) через `node-pty`**, читает её вывод, удаляет
ANSI/BOM и возвращает ответ. `node-pty` поставляется с готовыми бинарниками для
распространённых сочетаний Node/ОС и устанавливается автоматически при первом
использовании (в обычном случае инструментарий C++ не нужен).

Список моделей считывается вживую из интерактивного меню `/model` в agy и кэшируется по
отпечатку бинарника agy — он считывается заново автоматически, когда agy обновляется.

## Выбор модели

У agy **нет CLI-флага `--model`**, поэтому этот плагин выбирает модель, кратко и безопасно
переписывая `~/.gemini/antigravity-cli/settings.json`, а затем восстанавливая его. Это
делается под блокировкой и **устойчиво к сбоям** — ваши настройки никогда не остаются
повреждёнными, даже если запуск был прерван на полпути (оригинал сохраняется и
восстанавливается следующим запуском).

```bash
/agy:models                                  # see everything your account can run
/agy:model                                   # show the current default
/agy:model pro                               # set default to the strongest Gemini Pro
/agy:model flash                             # set default to Gemini Flash (fast, cheap)
/agy:model "Claude Opus 4.6 (Thinking)"      # default to a Claude model
/agy:ask --model flash  your question        # one-off override (doesn't change default)
```

- **Псевдонимы** (`pro`, `flash`, а также `pro-low`, `flash-medium`, …) **только для
  Gemini** и отслеживают актуальный список, поэтому `pro`/`flash` автоматически следуют за
  новейшим уровнем Gemini.
- Моделям **Claude / GPT-OSS** нужна **полная метка** — скопируйте её из `/agy:models`.
- Модель по умолчанию сохраняется в `~/.agy-jobs/config.json` — мгновенно, сохраняется
  между сессиями, без перезапуска терминала. Флаг `--model` для конкретного вызова всегда
  имеет приоритет над моделью по умолчанию.
- Каждый запуск сообщает, какая модель была **фактически использована** (считывается из
  собственного журнала agy, а не из самоотчёта модели — модели ненадёжны в назывании
  самих себя).

## Разрешения

- `ask` / `research` по умолчанию работают **только на чтение**; добавьте `--write`, чтобы
  разрешить правки.
- `rescue` по умолчанию **может записывать**; добавьте `--read-only` только для советов.
- `review` / `adversarial-review` **всегда только на чтение** — используйте `/agy:rescue`,
  чтобы действовать по их выводам.
- Запуски только на чтение передают agy флаг `--sandbox` (ограничения терминала): agy
  по-прежнему может читать и анализировать файлы, но системные/терминальные побочные
  эффекты блокируются.

## Замечания по платформам

- **Windows / Linux** — полностью протестировано (переключение моделей, считывание,
  устойчивое к сбоям восстановление — всё работает).
- **Нюанс Linux + SSH**: agy хранит данные входа в keyring рабочего стола, когда вы входите
  в графической сессии, но переключается на файловые токены, когда обнаруживает SSH-сессию
  (`SSH_CONNECTION`). Эти два хранилища не делятся состоянием, поэтому запуск плагина
  **через чистое SSH-соединение** может привести к ошибке «Authentication required», даже
  если вы вошли в систему на рабочем столе. Решения: войдите *внутри* SSH-сессии, **или**
  запускайте внутри сессии `tmux`/`screen`, начатой с рабочего стола (без `SSH_CONNECTION`
  в её окружении) — тогда agy читает данные входа рабочего стола как обычно. Это поведение
  самого agy CLI, а не ошибка плагина.

## ⚠️ Конфиденциальность — прочтите это

agy отправляет ваши запросы (а для `review` — и ваш diff кода) на **серверы Google**. **Не**
используйте его с секретами, учётными данными, приватными ключами или
конфиденциальной / неопубликованной работой, которую вы не можете передать третьей стороне.
Относитесь к нему как к любому другому облачному ИИ-сервису.

## Устранение неполадок

- **Команда `/plugin` не найдена** → ваш Claude Code слишком старый (ниже 2.1.143).
  Обновите приложение и перезапустите его (см. [Установка](#install)). Возможность
  использовать новую *модель* не означает, что приложение обновлено.
- **Установлено, но команды `/agy:*` не появляются** → запустите **`/reload-plugins`**;
  если они всё ещё не появляются, **полностью закройте и снова откройте** Claude Code.
  Новым файлам команд нужна перезагрузка/перезапуск для загрузки.
- **`/agy:setup` сообщает `agy binary: NOT FOUND`** → запустите `/agy:install` или задайте
  переменную окружения `AGY_BIN` с путём к исполняемому файлу agy.
- **`node-pty: UNAVAILABLE`** → разовая автоустановка не удалась; убедитесь, что Node.js +
  npm есть в PATH и есть сеть, затем повторно запустите `/agy:setup`.
- **Нет ответа / ошибка аутентификации** → один раз запустите `agy` в интерактивном режиме
  в терминале, чтобы войти.
- **Список моделей выглядит устаревшим после обновления agy** → `/agy:models --refresh` или
  `/agy:update`.

Не повторяйте попытки в цикле при сбое — устраните первопричину (аутентификация,
установка, сеть).
