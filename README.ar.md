# agy — Google Antigravity لـ Claude Code

استخدم **agy CLI** (Google Antigravity) كنموذج ثانٍ داخل Claude Code — وهو النظير الخاص بـ
`agy` للإضافة `codex`. اطرح الأسئلة، واحصل على آراء ثانية، وراجع
الـ diff الخاص بك، أو فوّض مهامًا قادرة على الكتابة، كل ذلك دون مغادرة Claude Code.

ما يجعله مثيرًا للاهتمام: **يستطيع agy تشغيل نماذج Gemini وClaude *و* GPT-OSS** خلف
واجهة CLI وحساب واحد. تُبرز هذه الإضافة ذلك — اختر أيًّا منها لكل استدعاء، أو عيّن
نموذجًا افتراضيًا، مباشرةً من Claude Code.

> ⚠️ **غير رسمية.** هذه إضافة مجتمعية، وليست تابعة لـ
> Google أو Anthropic ولا معتمدة منهما. "Antigravity" و"Gemini" و"Claude" و"Codex" ملك
> لأصحابها المعنيين.

## ما الذي تحصل عليه

| الأمر | ما الذي يفعله |
|---|---|
| `/agy:ask` | اطرح على agy سؤالًا لمرة واحدة (للقراءة فقط افتراضيًا) |
| `/agy:research` | اطلب من agy البحث وتجميع إجابة |
| `/agy:rescue` | فوّض مهمة/إصلاحًا — **قد يحرّر agy الملفات** |
| `/agy:review` | يراجع agy الـ diff المحلي في git (للقراءة فقط) |
| `/agy:adversarial-review` | مراجعة تنافسية صارمة للـ diff الخاص بك (للقراءة فقط) |
| `/agy:model` | عرض النموذج **الافتراضي** أو تعيينه |
| `/agy:models` | سرد **جميع** النماذج التي يمكن لحسابك استخدامها (Gemini / Claude / GPT-OSS) |
| `/agy:update` | تحديث agy CLI؛ يُحدّث قائمة النماذج |
| `/agy:setup` | فحص سلامة التكامل |
| `/agy:install` | تثبيت agy CLI (يسأل أولًا) |
| `/agy:status` `/agy:result` `/agy:cancel` | إدارة المهام في الخلفية |

## المتطلبات

- **Claude Code** (هذه إضافة له)
- **Node.js 18+** (بيئة التشغيل هي Node؛ ويُثبَّت `node-pty` تلقائيًا عند أول تشغيل)
- **agy CLI** (Google Antigravity). لا تملكه؟ شغّل `/agy:install` (يسأل
  أولًا)، أو ثبّته يدويًا من <https://antigravity.google>. بعد التثبيت، شغّل
  `agy` مرة واحدة بشكل تفاعلي لتسجيل الدخول.

جرى اختباره على **Windows** و**Linux** (x86_64). يُفترض أن يعمل على macOS (نفس مسارات
الشيفرة) لكنه غير مُختبَر. راجع *ملاحظات المنصة* أدناه للتنبيه الخاص بـ Linux/SSH.

## التثبيت

> **لا ترى الأمر `/plugin`؟** إصدار Claude Code لديك قديم جدًا — يحتاج `/plugin` إلى
> إصدار حديث (2.1.143+). حدّث Claude Code أولًا (تطبيق المتجر: التحديث عبر Microsoft
> Store / App store؛ سطر الأوامر: `claude update`)، ثم أعد تشغيله. كونك قادرًا على استخدام
> *نموذج* جديد مثل Opus 4.8 لا يعني **أن** تطبيقك محدَّث — فالنماذج تأتي من
> الخادم، أما ميزة `/plugin` فتأتي من التطبيق.

**الخطوة 1 — أضِف المتجر وثبّت الإضافة** (داخل Claude Code):

```bash
/plugin marketplace add alieuidsh/claude-agy-plugin
/plugin install agy@shuo-agy
```

**الخطوة 2 — أظهِر الأوامر. ⚠️ هذه الخطوة مطلوبة، وهي حيث يعلق الناس.**
الأوامر المثبَّتة حديثًا **لا** تظهر حتى تعيد التحميل أو التشغيل:

- شغّل **`/reload-plugins`**، **و**
- إذا ظلّت أوامر `/agy:*` لا تظهر (أو بعد *تحديث* إضافة)،
  **أغلق Claude Code تمامًا وأعد فتحه** (أغلق النافذة/التطبيق بالكامل، وليس مجرد
  التبويب). إعادة التحميل وحدها قد لا تكفي أحيانًا لملفات الأوامر الجديدة تمامًا.

**الخطوة 3 — فحص السلامة:**

```bash
/agy:setup     # verifies agy + node-pty + auth; auto-installs node-pty on first run
```

أول استدعاء لـ `/agy:*` على الإطلاق يستغرق ~15–20 ثانية (تثبيت node-pty لمرة واحدة + أول
استخراج لقائمة النماذج). هذا طبيعي — فهو مُخزَّن مؤقتًا بعد ذلك، والاستدعاءات اللاحقة سريعة.

أول أمر يُشغّل agy قد يستغرق ~15–20 ثانية (تثبيت node-pty لمرة واحدة + استخراج لقائمة
النماذج، وكلاهما مُخزَّن مؤقتًا بعد ذلك).

## كيف يعمل (ولماذا)

ينتج agy 1.0.x **مخرجات فقط عندما يكتشف وحدة طرفية حقيقية (TTY)** — أما استدعاء
`spawn()` العادي بلا واجهة فلا يُنتج شيئًا. لذلك تُشغّل هذه الإضافة agy داخل **وحدة طرفية
مُصطنعة (ConPTY) عبر `node-pty`**، وتقرأ مخرجاته، وتُزيل ANSI/BOM، وتُعيد
الإجابة. يأتي `node-pty` مع ثنائيات مبنية مسبقًا لتركيبات Node/نظام التشغيل الشائعة ويُثبَّت
تلقائيًا عند أول استخدام (دون الحاجة إلى سلسلة أدوات C++ في الحالة العادية).

تُستخرج قائمة النماذج مباشرةً من قائمة `/model` التفاعلية في agy وتُخزَّن مؤقتًا، مفهرسةً
وفق بصمة ثنائي agy — وتُعاد استخراجها تلقائيًا عند تحديث agy.

## اختيار نموذج

لا يملك agy **أي راية `--model` في سطر الأوامر**، لذلك تختار هذه الإضافة نموذجًا عبر إعادة
كتابة آمنة وموجزة لـ `~/.gemini/antigravity-cli/settings.json`، ثم استعادته. يتم ذلك
تحت قفل وهو **آمن ضد الانهيار** — فإعداداتك لا تُترك أبدًا تالفة، حتى لو قُتل
التشغيل في منتصفه (يُحفظ الأصل ويُسترجَع في التشغيل التالي).

```bash
/agy:models                                  # see everything your account can run
/agy:model                                   # show the current default
/agy:model pro                               # set default to the strongest Gemini Pro
/agy:model flash                             # set default to Gemini Flash (fast, cheap)
/agy:model "Claude Opus 4.6 (Thinking)"      # default to a Claude model
/agy:ask --model flash  your question        # one-off override (doesn't change default)
```

- **الأسماء المستعارة** (`pro`، `flash`، إضافةً إلى `pro-low`، `flash-medium`، …) خاصة
  بـ **Gemini فقط** وتتابع القائمة الحية، لذا يتبع `pro`/`flash` أحدث فئة من Gemini تلقائيًا.
- تحتاج نماذج **Claude / GPT-OSS** إلى **التسمية الكاملة** — انسخها من `/agy:models`.
- يُحفظ الافتراضي في `~/.agy-jobs/config.json` — فوري، ويثبت عبر الجلسات،
  دون إعادة تشغيل الطرفية. أما `--model` لكل استدعاء فيتغلّب دائمًا على الافتراضي.
- يُبلّغ كل تشغيل عن النموذج **المستخدَم فعليًا** (مقروءًا من سجل agy نفسه، لا من
  تقرير النموذج الذاتي — فالنماذج غير موثوقة في تسمية أنفسها).

## الأذونات

- `ask` / `research` للقراءة فقط **افتراضيًا**؛ أضِف `--write` للسماح بالتعديلات.
- `rescue` **قادر على الكتابة** افتراضيًا؛ أضِف `--read-only` للحصول على نصيحة فقط.
- `review` / `adversarial-review` **دائمًا للقراءة فقط** — استخدم `/agy:rescue` للتصرّف بناءً
  على النتائج.
- تمرّر عمليات التشغيل للقراءة فقط راية agy `--sandbox` (قيود الطرفية): لا يزال بإمكان agy
  قراءة الملفات وتحليلها، لكن الآثار الجانبية للنظام/الطرفية محجوبة.

## ملاحظات المنصة

- **Windows / Linux** — مُختبَران بالكامل (تبديل النماذج والاستخراج والاستعادة الآمنة ضد الانهيار، جميعها تعمل).
- **مشكلة Linux + SSH**: يخزّن agy تسجيل دخولك في keyring سطح المكتب عندما تسجّل الدخول في
  جلسة رسومية، لكنه يتحوّل إلى رموز قائمة على الملفات عندما يكتشف جلسة SSH
  (`SSH_CONNECTION`). الاثنان لا يتشاركان الحالة، لذا فإن تشغيل الإضافة **عبر اتصال SSH
  مجرّد** قد يصطدم بـ "Authentication required" حتى وإن كنت مسجَّل الدخول على
  سطح المكتب. الحلول: سجّل الدخول *داخل* جلسة SSH، **أو** شغّل داخل جلسة `tmux`/`screen`
  بُدئت من سطح المكتب (لا يوجد `SSH_CONNECTION` في بيئتها) —
  عندها يقرأ agy تسجيل دخول سطح المكتب بشكل طبيعي. هذا سلوك خاص بـ agy CLI، وليس خللًا في الإضافة.

## استكشاف الأخطاء وإصلاحها

- **الأمر `/plugin` غير موجود** → إصدار Claude Code لديك قديم جدًا (أقل من 2.1.143). حدّث
  التطبيق وأعد تشغيله (انظر [التثبيت](#install)). كونك قادرًا على استخدام *نموذج* جديد لا
  يعني أن التطبيق محدَّث.
- **مثبَّتة، لكن أوامر `/agy:*` لا تظهر** → شغّل **`/reload-plugins`**؛ إذا
  ظلّت لا تظهر، **أغلق Claude Code تمامًا وأعد فتحه**. تحتاج ملفات الأوامر الجديدة إلى
  إعادة تحميل/إعادة تشغيل لتُحمَّل.
- **يقول `/agy:setup`: `agy binary: NOT FOUND`** → شغّل `/agy:install`، أو عيّن متغير
  البيئة `AGY_BIN` إلى مسار ملف agy التنفيذي.
- **`node-pty: UNAVAILABLE`** → فشل التثبيت التلقائي لمرة واحدة؛ تأكد من وجود Node.js + npm
  على PATH ومن توفر شبكة، ثم أعد تشغيل `/agy:setup`.
- **لا توجد إجابة / خطأ مصادقة** → شغّل `agy` مرة واحدة بشكل تفاعلي في طرفية لتسجيل الدخول.
- **تبدو قائمة النماذج قديمة بعد تحديث agy** → `/agy:models --refresh` أو `/agy:update`.

لا تُعِد المحاولة في حلقة عند الفشل — أصلح السبب الجذري (المصادقة، التثبيت، الشبكة).
